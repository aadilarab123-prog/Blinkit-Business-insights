Blinkit Grocery Sales & Performance Analysis (Power BI Dashboard)
📌 Project Overview
This project presents a comprehensive, data-driven analysis of Blinkit's sales performance, customer satisfaction, and inventory distribution. The core objective is to uncover strategic insights and identify performance optimization opportunities across various outlet tiers, locations, and product segments using an interactive Power BI dashboard.

The dataset analyzed includes data from BlinkIT Grocery Data

📊 Business Requirements & Key Metrics
The analysis is anchored around four foundational Key Performance Indicators (KPIs) to monitor global performance:

Total Sales: The overall revenue generated from all grocery items sold.

Average Sales: The average revenue generated per transaction/sale.

Number of Items: The total count of distinct product SKUs sold.

Average Rating: The average customer satisfaction rating scored across items.

🔍 Granular Insights & Charting Framework
The dashboard approaches the data from multiple analytical angles to answer specific operational questions:

1. Product Segment Analysis
Total Sales by Fat Content: Analyzes the operational impact of Low Fat vs. Regular fat variants on total revenue, detailing how secondary metrics (Average Sales, Item Count, and Ratings) shift based on product health attributes.

Total Sales by Item Type: Evaluates the relative performance of distinct grocery categories (Fruits, Vegetables, Dairy, Snacks, etc.) to isolate high-velocity revenue drivers from slower inventory items.

2. Operational & Outlet Tier Analysis
Fat Content Segments Across Outlets: A comparative look at total sales across varying outlet formats, segmented cleanly by product fat content.

Total Sales by Outlet Establishment Year: Explores the correlation between the operational age/maturity of a store location and its cumulative revenue.

Percentage of Sales by Outlet Size: Uses structured visual types to analyze the impact of physical store footprints (Small, Medium, Large) on sales volume.

Sales Distribution by Outlet Location: Breaks down the geographic performance across Tier 1, Tier 2, and Tier 3 city clusters.

3. Holistic Matrix Overview
All Metrics Tiered by Outlet Type: A final comprehensive matrix breaking down all core KPIs simultaneously by structural outlet types (e.g., Supermarket Type 1, Type 2, Type 3, Grocery Stores) to understand exact unit economics.

🛠️ Tech Stack & Tools Used
Data Source: BlinkIT Grocery Data

BI Platform: Microsoft Power BI Desktop

Data Transformation: Power Query (ETL, data cleaning, renaming text values, structural type casting)

Modeling Language: DAX (Data Analysis Expressions) for explicit dynamic measures (Total Sales, Average Sales, Item Counts, etc.)
